# Eng AI chat

> Search internal company knowledge grounded in application context.

## What it does

This skill searches internal knowledge bases with application-aware context. It discovers relevant apps from your current repository, then searches across AI-generated summaries, developer docs, code search results, and internal documentation to answer questions about services, infrastructure, integrations, and platform guidance. Results are synthesized from multiple sources for comprehensive answers.

## Usage

Use when you need company-specific information about apps, services, infrastructure, or platform guidance. The skill can auto-discover relevant apps from your repo or accept explicit app names.

## Examples

- "How do I add a second deployment to my service?"
- "How can I integrate with the session validation service?"
- "What's the recommended way to set up background jobs for my app?"

## Why it was created

Internal engineering knowledge is spread across docs, code, and tribal knowledge. This skill provides a single entry point that searches across all sources with app-specific context, saving engineers from manually hunting through documentation.
