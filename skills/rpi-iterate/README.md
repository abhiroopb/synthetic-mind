# RPI Iterate

> Update existing RPI implementation plans based on feedback, new requirements, or discoveries during implementation.

## What it does

RPI Iterate is the iteration phase of the RPI methodology. It takes an existing implementation plan and modifies it based on user feedback — adding phases, adjusting scope, updating success criteria, or refining the approach. It reads the full existing plan, researches the codebase if needed to validate changes, confirms understanding before making edits, and makes surgical updates while preserving good content that doesn't need changing.

## Usage

Invoke when an existing RPI plan needs adjustments based on feedback, new requirements, or things discovered during implementation.

**Trigger phrases:**
- "Update the plan based on this feedback"
- "Add a migration phase to the plan"
- "Adjust the scope of the implementation plan"
- "Refine the success criteria"

## Examples

- `"Update the plan at thoughts/shared/plans/2025-03-01-feature.md to add a rollback phase"`
- `"Adjust the scope of the checkout plan to exclude mobile"`
- `"Split phase 2 of the plan into two separate phases"`

## Why it was created

Implementation plans rarely survive first contact with reality unchanged. This skill provides a structured way to update plans based on feedback while maintaining quality standards — making precise edits rather than wholesale rewrites, and researching the codebase to validate feasibility of changes.
