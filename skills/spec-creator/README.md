# Spec creator

> Create and iterate on product requirement specs using a structured template, pulling from various data sources.

## What it does

Helps product managers write comprehensive, concise product requirement documents by gathering context from multiple sources and structuring them into a proven template. It covers everything from problem statement and user workflows to competitive analysis, phasing, and success metrics. Specs can be synced to project tracking tools to keep documentation aligned with execution.

## Usage

Invoke when you need to create or iterate on a product spec. Trigger phrases:

- "Create a spec for [feature]"
- "Write a product spec"
- "Draft a requirements spec for [feature area]"

## Examples

- `"Create a spec for offline payments support on POS"`
- `"Iterate on the checkout redesign spec — add competitive analysis"`
- `"Draft a spec for the new tipping flow and sync it to our project tracker"`

## Why it was created

Starting a spec from scratch is slow and inconsistent. This skill enforces a repeatable template that captures the right level of detail — goals, user workflows, competitive context, phasing, and metrics — so PMs can focus on thinking rather than formatting.
