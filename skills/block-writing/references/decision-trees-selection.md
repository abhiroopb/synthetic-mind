# Decision Trees: Selection Controls

## Selection Control Content Decision Tree

```
Writing selection control content? → What type of selection?

├── Checkbox → Describe what happens when selected
│   Don't use "check" or "select" in labels; don't write as questions; parallel phrasing in groups
│   Legal agreements: first person ("I agree...")
│   Cash App: "Push notifications", "Auto-invest"
│   Square: "Email receipts", "Inventory alerts"
│   Legal: "I agree to the Terms of Service"
│
├── Radio button → Keep short and scannable
│   Use parallel structures; logical ordering (most common first); avoid yes/no (use checkbox/toggle)
│   Payment: "Cash App Card", "Bank account", "Bitcoin"
│   Business type: "Retail store", "Restaurant", "Service business"
│
├── Toggle → Describe what will be enabled when "on"
│   Don't use "turn on" or "enable" in labels; use secondary description if needed
│   Cash App: "Push notifications", "Paper statements"
│   Square: "Automatic tax calculation", "Receipt printing"
│
└── Segmented control → Binary content switching
    "Light mode" / "Dark mode" | "List view" / "Grid view"
    Keep parallel structure and length
```

### Selection Control Context Decision

```
Where will the control appear?

├── Settings list or preferences → Clear, descriptive labels; avoid jargon; consistent across similar settings
│
├── Form or signup flow → Action-oriented; clear about consequences; first person for agreements
│
└── Quick selection interface → Scannable, parallel options; similar length; familiar terminology
```
