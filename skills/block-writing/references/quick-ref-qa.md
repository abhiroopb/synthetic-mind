# Component Content QA Checklist

## Before Publishing

**Format and Style**:
- [ ] Sentence case applied consistently
- [ ] Character limits respected for component type
- [ ] No end punctuation on labels/buttons
- [ ] Proper punctuation on complete sentences

**Voice and Clarity**:
- [ ] Block voice principles reflected appropriately
- [ ] Product-specific terminology used correctly
- [ ] Content makes sense without visual context
- [ ] Language is simple and accessible (7th-8th grade)

**Component Standards**:
- [ ] Follows established pattern for component type
- [ ] Action-oriented button text with specific verbs
- [ ] Clear, helpful error messages with solutions
- [ ] Consistent terminology across similar components

**Accessibility**:
- [ ] Works with screen readers
- [ ] Uses inclusive, non-discriminatory language
- [ ] Avoids directional or color-dependent instructions

## Common Mistakes to Avoid

- ❌ Title Case ("Save Your Changes")
- ❌ End punctuation on buttons or labels ("Save changes.")
- ❌ Generic button labels ("Submit", "OK", "Done")
- ❌ Vague error messages ("Something went wrong")
- ❌ Blaming users in errors ("You entered invalid data")
- ❌ Technical jargon instead of plain language
- ❌ Exceeding character limits for component type
- ❌ Content that only makes sense with visual cues

## Emergency Quick Fixes

1. Check capitalization — should be sentence case
2. Remove end punctuation from buttons/labels
3. Make button text action-oriented with specific verbs
4. Add solutions to error messages
5. Simplify language to 7th-8th grade level
6. Remove blame language from errors
7. Make labels descriptive without visual context
8. Use consistent terminology throughout
9. Check character limits for component type
10. Apply appropriate Block voice principle for context

**Most common quick wins**:
- "Submit" → specific action like "Save changes"
- "Error" → specific problem + solution
- "This field is required" → "[Field name] is required"
- "Click here" → descriptive action
- Title case → sentence case
- Period on button → remove
- Missing "Try again" button on error → add it
