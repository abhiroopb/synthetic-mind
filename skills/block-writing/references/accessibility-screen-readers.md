# Accessibility: Screen Readers and Text Alternatives

## Write for Screen Readers
Screen readers read content aloud to users with visual impairments.

**Best practices**:
- Use clear, descriptive headings in logical order (H1 → H2 → H3)
- Provide text alternatives for images, icons, and visual elements
- Write descriptive link text (not "click here")
- Use proper semantic structure (lists, headings, landmarks)
- Avoid relying on visual formatting alone to convey meaning

**Link text**:
- ✅ "View your account settings"
- ✅ "Learn about direct deposit"
- ❌ "Click here"
- ❌ "Learn more" (without context)

**Image alt text**:
- ✅ alt="Brand B Card, a black debit card with Brand B logo"
- ❌ alt="card" (too vague)
- ❌ alt="image123.jpg" (not descriptive)

**Button labels**:
- ✅ "Close dialog" or "Close"
- ❌ "X" (screen reader would say "X button")

## Provide Text Alternatives
Every non-text element should have a text alternative.

**What needs alternatives**:
- Images and photos: Alt text
- Icons: Accessible labels
- Charts and graphs: Text description or data table
- Videos: Captions and transcripts
- Audio: Transcripts
