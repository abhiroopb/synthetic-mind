# Accessibility: Readability, Hierarchy, and Color

## Target Appropriate Reading Level

**General audiences**: 7th-8th grade reading level
**Professional tools**: 9th-10th grade reading level

**How to achieve this**:
- Use common, everyday words
- Keep sentences short (8-14 words for maximum comprehension)
- Break up complex ideas into multiple sentences
- Define technical terms when you must use them
- Use active voice

**7th-8th grade level**:
- ✅ "Connect your bank to add money automatically"
- ❌ "Establish a connection to your financial institution to facilitate automated fund transfers"

**Technical terms defined**:
- ✅ "Enable two-factor authentication (2FA). This adds an extra security step when you sign in."
- ❌ "Enable 2FA" (no definition)

## Use Clear Hierarchy
Structure content logically with clear headings.

**Best practices**:
- Only one H1 per page
- Don't skip heading levels (H1 → H2, not H1 → H3)
- Use headings to organize content, not just for visual styling
- Make headings descriptive of the content that follows

## Don't Rely on Color Alone
Never use color as the only way to convey information.

**Error states**:
- ✅ Red color + error icon + error message text
- ❌ Red color only

**Status indicators**:
- ✅ Green dot + "Active" label
- ❌ Green dot only (colorblind users can't distinguish)

**Links**:
- ✅ Underlined + colored text
- ❌ Colored text only (not enough contrast from body text)
