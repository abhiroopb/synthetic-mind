# CF1 Knowledge Article Guidelines (Salesforce)

For Cash App customer advocacy teams writing internal Salesforce Knowledge Articles.

## Naming Conventions
Format: `Team/Product - Issue - Sub-Issue or Region`
- ✅ "Cash App Card - Digital Wallet"
- ✅ "Cash App Card Disputes - Mobile Pay Fraud (Reg E)"
- Channel-first when channel-specific: "Voice - Cash App Card Disputes - Submitting IRF"

## Copy-Paste from Google Docs
Text copied from Google Docs won't be properly formatted in CF1.
- Paste text, select it, click "Remove Formatting" button
- Or use Shift+Cmd+V to paste without formatting
- Never paste images from Google Docs—download and re-upload to CF1 instead

## Links
**Internal (CF1 articles)**: Always use Smart Links
- Click Smart Link button → search for article → select it
- Set Target: Same Window (_self) / Salesforce Target: [blank]
- ⚠️ Incorrect Smart Links cause Omnichannel to pull extra support cases
- You can link to Draft articles (link updates when published)

**External links**: Use normal hyperlink (Cmd+K); set to open in New Window (_blank)
- Add a heads-up when linking outside CF1

## Structure and Word Counts
- Large topics: create a landing page article + linked topic articles
- **Goal**: <4,000 words per article; **Max**: 10,000 words
- CF1 character limit: 131,072 (includes links, graphics, formatting—cannot increase)
- All headings: sentence case

## Channel Workflows
Structure articles by channel when workflows differ significantly.
- Create **separate article** when ≥1/4 of workflow is unique to a channel
- Create **section within article** when <1/4 of workflow is channel-specific
- Always list channels in order: Voice, then Messaging, then Email

## Formatting Tips
- Include a table of contents with jump links (anchor links) at the top
- Break paragraphs longer than 3 sentences; use subheadings throughout
- Bold, italics, or highlights for key information
- Use bullets and numbered lists to break up text blocks

## Publishing
- You can schedule articles for a future date/time
- **Can't cancel** a draft scheduled to publish—must change date or delete/republish
- Versions field shows last 30 published versions and drafts
