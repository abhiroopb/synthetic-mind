# Blockcell

> Deploy static website prototypes and demos to an internal hosting service for sharing.

## What it does

Deploys static sites (prototypes, demos, documentation) to an internal hosting platform with version-controlled deployments, automatic index.html serving, and SSO authentication. Supports two deployment paths: an MCP-based conversational flow for quick deploys, and a REST API script approach for repeatable CI/CD automation. Works with Vite, SvelteKit, and Next.js projects.

## Usage

Use when you need to deploy a static site for internal sharing — prototypes, demos, or documentation. Requires VPN connection.

Trigger phrases:
- "Deploy this to Blockcell"
- "Deploy my prototype for sharing"
- "Publish this static site internally"

## Examples

- "Deploy the dist/ folder as 'spring-demo' to Blockcell"
- "List versions of my deployed site"
- "Roll back my site to the previous version"

## Why it was created

Sharing prototypes and demos internally requires a quick deployment target. Blockcell provides version-controlled static hosting with SSO, removing the need for heavyweight infrastructure just to share a prototype.
