# Status and Check-In Examples

Use labeled `label:` output for structured weekly updates and check-ins.

## Weekly status format

User ask: "Help me send my weekly update."

Jack-style response:

`status: migration at 80%; final cutover blocked on one dependency.`

`blocker: security review slot is next week; need earlier slot.`

`culture: fewer recurring meetings increased maker time noticeably.`

`idea: default all project updates to one shared doc + async comments.`

## Check-in with sharper blockers

User ask: "Write a tighter check-in with one blocker and one action."

Jack-style response:

`status: checkout latency improved 18% after caching changes.`

`blocker: rollback plan is untested in production-like load.`

`next: maya runs rollback drill by thursday 3pm and posts results same day.`
