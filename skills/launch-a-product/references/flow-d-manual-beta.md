# Flow D: Manual Beta (Bespoke Rollouts)

Load when PM selects "Manual Beta/GA" and chooses Beta. Use for Early Feature Access (EFA), bespoke rollouts, or features that need more white-glove service than the standard bundle process.

## D1. Verify Project Description

- [ ] Show the current Project Description
- [ ] Ask: "Is this still accurate, or should we update it?"
- [ ] If update needed, revise using GTM Messaging Toolkit and GTM Guide for PMs

## D2. Validate All Fields

Validate all 38 fields from `field-checklists.md`, with additional beta-specific value checks:

- **Launch Type** should be "Closed Beta" or "Open Beta"
- **Rollout Start Date** must be set
- **Legal, Regulatory Comms Required** should reflect beta comms plan
- **Project Status** should reflect beta stage
- **Weekly Status** and **Update** should be current

Flag any fields whose values don't match beta expectations.

## D3. Identify Interested Sellers

Search Slack for seller interest using the channels listed in SKILL.md. Search for the project/feature name across all channels. For each relevant message capture: seller name, channel, context/quote, account manager, date.

## D4. Define Rollout Plan

**Step 1:** Show Platform and Locale from Blueprint, plus interested sellers from D3.

**Step 2:** Ask the PM:

```text
1) How many sellers have you already been working with closely?
2) Is there a natural way to segment for a phased rollout?
3) What's your biggest risk? (technical stability / seller confusion / operational load / business risk / other)
4) How quickly do you want to ramp? (cautious / moderate / aggressive)
5) Are there specific sellers that MUST be in the first wave?
6) Are there sellers to explicitly EXCLUDE or delay?
```

**Step 3:** Build the rollout plan from answers. Don't assume wave count — could be 2 to 5+.

**Step 4:** For each wave define: name/rationale, seller criteria, approximate count, duration, success gate, rollback criteria.

**Step 5:** Present for review. After approval, ask: **"Would you like me to put this rollout plan into a Google Doc for easy sharing?"**

## D5. Draft Comms

**Internal CS/AM Prep Materials:**
- Briefing doc: what the feature does, who it's for, known limitations
- FAQ: common questions CS/AM will receive
- Rollout schedule

**Seller-Facing Beta Announcement:**
- Beta invitation language (tailored per wave)
- What's included / what's not yet available
- How to provide feedback
- Timeline expectations

After presenting, ask: **"Would you like me to put these comms into a Google Doc for easy sharing?"**

Continue to `flow-d-manual-beta-wrapup.md` for steps D6-D8.
