# Flow C: Pre-Launch Wrap-Up (C4-C7)

Load after completing C3 (Verify In Progress Fields) in `flow-c-pre-launch.md`.

## C4. Go/No-Go Preparation

Help the PM prepare for Go/No-Go review (~3 weeks before 100% GA):

- [ ] Confirm all Blueprint fields are complete and accurate
- [ ] Confirm feature is stable (ask about test results, known issues)
- [ ] Confirm LaunchDarkly flag is set up and linked in Resources
- [ ] Confirm Linear project is linked in Resources
- [ ] Review Project Description for clarity — this will power automated comms

**Remind the PM:** "If you miss Go/No-Go, you miss the bundle. No exceptions."

## C5. Verify Automated Comms Inputs

Under the unified release process, these are triggered automatically from Blueprint:
- **Feature Log / Twitter / Reddit** → biweekly at launch
- **Seller email + in-product alerts** → 1 week pre-launch
- **AM-managed seller enablement** → 2 weeks post-launch (after holdback)

Help the PM verify the fields that feed these automated comms:
- [ ] Project Description — clear, seller-value-focused, no jargon
- [ ] Share Externally — set correctly
- [ ] Demo Video URL — linked if available
- [ ] Platform / Locale — accurate for targeting

**Note:** Images/visuals for automated comms are TBD.

## C6. Update Airtable

- [ ] Show PM a diff of all proposed changes
- [ ] Get explicit PM approval
- [ ] Execute updates

After presenting, ask: **"Would you like me to put this readiness summary into a Google Doc for easy sharing?"**
