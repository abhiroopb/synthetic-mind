# Brand A Staff Daily Tip Payouts - Data Guide

## Business Context

Daily Tip Payouts is a Brand A Staff feature launched in Fall 2025 that enables merchants to pay out employee tips through Brand B integration. This feature allows employees to receive their tips quickly via Brand B instead of waiting for traditional payroll cycles.

**Key Business Facts:**
- Launched Fall 2025 as a Staff product feature
- Integrates Brand A tip collection with Brand B payouts
- Merchants can pay out tips for specific time periods (daily, custom ranges)
- Employees must link their Brand B account to receive payouts
- Tips can be attributed to specific employees or remain unattributed
- Primary dashboard: go/YOUR_INTERNAL_LINK

## Data Hierarchy

The tip payout system has three levels of granularity:

BATCH (merchant-level operation)
├── TIP PAYMENT (unit 1)
│ ├── Tip Payment Transaction (employee A)
│ ├── Tip Payment Transaction (employee B)
│ └── Tip Payment Transaction (employee C)
├── TIP PAYMENT (unit 2)
│ ├── Tip Payment Transaction (employee D)
│ └── Tip Payment Transaction (employee E)
└── TIP PAYMENT (unit 3)
└── Tip Payment Transaction (employee F)


**BATCH:** Merchant's complete tip payout operation for a time period (all units together)
**TIP PAYMENT:** One unit's tip payouts for a specific time period
**TIP PAYMENT TRANSACTION:** Individual payout to one employee

## Primary Tables

### YOUR_ANALYTICS_SCHEMA.FACT_TIP_PAYMENT_TRANSACTIONS (VERIFIED)
**Purpose:** Primary analytics table for employee-level tip payouts
**Grain:** Transaction-level (one row per employee tip payment)
**Primary Key:** TIP_PAYMENT_TRANSACTION_ID
**Table Owners:** LDAP_B, LDAP_A

**Status Fields:**
- `TRANSACTION_STATUS_READABLE` (text): Values: 'COMPLETED', 'DRAFT', 'ERRORED', 'VOIDED'
- `IS_COMPLETED` (boolean): TRUE if transaction completed successfully
- `PAYMENT_STATUS_READABLE` (text): Values: 'COMPLETED', 'DRAFT', 'ERRORED', 'PARTIALLY_COMPLETED'
- `PAYMENT_ERROR_REASON` (text): Error description if applicable

### YOUR_ANALYTICS_SCHEMA.FACT_TIP_PAYMENTS (UNVERIFIED)
**Purpose:** Payment-level analytics for unit tip payouts
**Grain:** Payment-level (one row per unit-level payout)
**Primary Key:** TIP_PAYMENT_TOKEN
**Table Owners:** LDAP_B

### YOUR_ANALYTICS_SCHEMA.TIP_PAYOUT_MERCHANT_LIFETIME_SUMMARY (VERIFIED)
**Purpose:** Merchant-level lifetime metrics for tip payout adoption
**Grain:** One row per merchant
**Primary Key:** CUSTOMER_ID
**Table Owners:** LDAP_B, LDAP_C, LDAP_A

### YOUR_ANALYTICS_SCHEMA.TIP_PAYOUT_EMPLOYEE_LIFETIME_SUMMARY (UNVERIFIED)
**Purpose:** Employee-level lifetime metrics and Brand B integration
**Grain:** One row per employee
**Primary Key:** EMPLOYEE_TOKEN
**Table Owners:** LDAP_B

Important Business Rules
Submitted_At is Key Date: Use SUBMITTED_AT field to filter for when daily tip payout occurred, not created_at

Status Codes:

Transaction Status: COMPLETED, DRAFT, ERRORED, VOIDED
Payment Status: COMPLETED, DRAFT, ERRORED, PARTIALLY_COMPLETED
Money Conversion: All amount fields are in cents - divide by 100 to get dollars

Batch Identification: BATCH_ID = customer_id + created_at timestamp pattern

Mixed Grain Design: Transaction table has both transaction-level and payment-level fields (payment_ prefix)

Tip Attribution: Tips can be:

Attributed to specific employees
Unattributed (no employee assigned)
Attributed to linked employees (Brand B connected)
Multi-Unit Merchants: One batch can contain multiple tip payments (one per unit)

Brand B Linking: Employees must link Brand B to receive payouts

Feature Scope: This data only includes merchants who adopted daily tip payouts feature. For complete tip analysis, use app_pa.app_sqs.sqs_feature_usage

Common Filters
Date Filters (ALWAYS USE SUBMITTED_AT):


-- Last 30 days
WHERE submitted_at >= CURRENT_DATE - 30

-- Specific date range
WHERE submitted_at BETWEEN '2025-01-01' AND '2025-01-31'

-- For payment table, use tip_queried_start_date
WHERE tip_queried_start_date >= '2025-01-01'
Status Filters:


-- Completed transactions only
WHERE is_completed = TRUE

-- Completed payments only
WHERE payment_status_readable = 'COMPLETED'

-- Exclude errors
WHERE transaction_status_readable != 'ERRORED'
Attribution Filters:


-- Only attributed tips
WHERE tips_match_payout = TRUE

-- Tips to linked employees
JOIN app_pa.app_sqs.tip_payout_employee_lifetime_summary els
  ON t.employee_token = els.employee_token
WHERE els.brand_b_customer_id IS NOT NULL
Key Metrics & Definitions
Adoption Rate: % of eligible merchants who have adopted daily tip payouts
Attribution Rate: % of collected tips attributed to specific employees
Linked Employee Rate: % of employees who have linked Brand B
Batch Completion Rate: % of batches that reach COMPLETED status
Average Tips Per Employee: Total tip amount / employee count
Payout Match Rate: % of transactions where payout equals original tip amount
