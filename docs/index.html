Persistent Memory — amp-mem

SQLite + FTS5 memory that persists across sessions. No vector embeddings — the LLM is the semantic engine.

🏗️ Architecture

~/.amp/memory/
├── amp-mem.db        # SQLite (WAL mode)
├── schema.sql        # Database schema
├── knowledge.md      # Plain-text log
└── web/              # Web viewer

~/bin/amp-mem          # CLI (16 commands)
~/.agents/skills/
├── memory/SKILL.md
└── kb-distill/SKILL.md

💡 Design Decisions

  • No vector embeddings — the LLM is the semantic engine
  • SQLite + FTS5 — porter stemming, single file, WAL mode
  • Progressive disclosure — search → timeline → detail
  • On-demand web viewer — no persistent service
  • Idempotent init — safe to run every session
  • Distillation — compresses observations into knowledge

⌨️ CLI Commands

🏷️ Observation Types

Tips

💬 Just Ask Naturally

With auto-pilot, you don't need skill names.

❌ "Use the slack skill to search for messages"
✅ "Search Slack for release updates this week"

🔬 Use RPI for Big Tasks

For complex work, use Research → Plan → Implement. Each phase runs in its own session so context stays fresh.

🧠 Let Amp Remember

With amp-mem, Amp remembers across sessions — workflows, decisions, team structures. Teach it once, it remembers forever.

⚙️ Configure AGENTS.md

Your ~/AGENTS.md is the master config — identity, preferences, automated behaviors. Your personal OS for Amp.
