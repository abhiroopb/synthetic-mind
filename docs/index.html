How Skills Work

Skills are markdown "recipes" that teach Amp to use your tools

1

You ask

"Search Slack for release updates this week"

2

Amp loads the skill

Auto-pilot routes to the right skill automatically

3

Skill provides instructions

CLI tools, auth flow, output format — all defined in SKILL.md

4

Amp executes

Runs commands, processes results, returns them to you

Persistent Memory with amp-mem

SQLite + FTS5 memory that persists across sessions — no vector embeddings needed

🏗️ Architecture

~/.amp/memory/
├── amp-mem.db        # SQLite (WAL mode)
├── schema.sql        # Database schema
├── knowledge.md      # Plain-text log
└── web/
    ├── index.html    # Web viewer
    └── serve.py      # Local server

~/bin/amp-mem          # CLI (16 commands)

~/.agents/skills/
├── memory/SKILL.md
└── kb-distill/SKILL.md

💡 Key Design Decisions

  • No vector embeddings — the LLM itself is the semantic engine
  • SQLite + FTS5 — porter stemming, single file, WAL mode
  • Progressive disclosure — search → timeline → detail
  • On-demand web viewer — no persistent service needed
  • Idempotent init — safe to run every session start
  • Distillation — compresses raw observations into knowledge

Tips

Get the most out of your Amp setup

💬 Just Ask Naturally

With auto-pilot, you don't need skill names. Just describe what you want.

❌ "Use the slack skill to search for messages"
✅ "Search Slack for release updates this week"

🔬 Use RPI for Big Tasks

For complex work (refactors, new features, migrations), use Research → Plan → Implement. Each phase runs in its own session so context stays fresh.

🧠 Let Amp Remember

With amp-mem, Amp remembers context across sessions — workflows, decisions, team structures. You teach it once, it remembers forever.

⚙️ Configure AGENTS.md

Your ~/AGENTS.md is the master config — identity, preferences, automated behaviors. Treat it as your personal operating system for Amp.

Quick Setup

Get up and running in minutes

📋 Prerequisites

  • Amp CLI — install via ampcode.com
  • GitHub CLI — brew install gh && gh auth login
  • uv — curl -LsSf https://astral.sh/uv/install.sh | sh

📦 Install Skills

# Install a single skill
amp skill add your-org/skills/slack

# Install with overwrite (updates)
amp skill add your-org/skills/slack --overwrite

# Remove a skill
amp skill remove slack
