Prototype Builder

skill View on GitHub
Scaffold and build interactive HTML prototypes for product features using a design system component library.

What it does

Prototype Builder helps create interactive prototypes that bring feature specs to life for stakeholder review and user testing. It reads the feature's spec, asks clarifying questions about scope and fidelity, scaffolds a prototype from a starter template, and iterates based on feedback. Prototypes are built as static HTML/CSS/JS files that can be opened directly in a browser — no build step required.

Usage

Invoke when you need to create an interactive prototype for a product feature. The skill reads the feature's spec and walks through scope, fidelity level, and framework choices before building.

Trigger phrases:

  • "Build a prototype for this feature"
  • "Create an interactive demo of the checkout flow"
  • "Prototype the new settings screen"

Examples

  • "Build a mid-fi prototype of the settings configuration screen"
  • "Create an end-to-end prototype of the checkout workflow"
  • "Prototype two variants of the payment confirmation modal for comparison"

Why it was created

Stakeholder reviews and user testing benefit enormously from interactive prototypes, but building them from scratch is slow. This skill accelerates prototyping by providing structure, templates, and a consistent design system so prototypes look production-realistic with minimal effort.
