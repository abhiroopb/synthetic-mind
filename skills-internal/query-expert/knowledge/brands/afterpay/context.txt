Table Types:
- Analytics: These tables are curated for analysis and should be your primary uses. 
- Production: These tables are replicated data from Square's APIs and Product. This data is not curated for analysis. Only use Production tables if there is no similar analytics table 
- Event: These tables track user initiated events such as clicks, taps, and views on Block's web and mobile software products. These tables contain a variety of events and require explicit filters on event descriptions and dates. 

Date Filters: 
- Always apply a date filter to a query. 
- Use the query_expert_search tool to find the most common date filter for that table 
- If the user does not request a date filter, suggest one
- Dimension tables are an exception. Often they do not have date fields to filter on.

Column Values: 
- Do not invent the values of a column. Always ensure accuracy based on query_expert_search tool, from the user input or from the 
- Use query_expert_search tool to find potential values for a particular column on a table by using the table_names parameter and the column name as the search text with other relevant keywords 
- If column values are unknown, execute a simple query selecting the distinct column_name with a short date period. Or an aggreagte function
    - For example, "SELECT DISTINCT feature_key, jtbd, sub_jtbd FROM APP_PA.APP_SQS.SQS_FEATURE_DIM" 
    - For example, "SELECT merchant_receipt_country_code, count(distinct merchant_token) as merchants from app_bi.hexagon.vdim_merchant group by 1 order by 2 desc Limit 100" 
- If a dimension table exists to discover column values, then use that one over a larger, transactional or aggregated table 
    - For example, to understand PRODUCT_NAME values, use APP_BI.HEXAGON.VDIM_PRODUCT instead of querying for PRODUCT_NAME within APP_BI.HEXAGON.VFACT_SUBSCRIPTION_STATES_EXPANDED

Best Schemas:
- AP_CUR_BI_G.CURATED_ANALYTICS_GREEN is the best schema for analytics and data analysis on Afterpay data 

Shein: 
- This is a large merchant that uses afterpay 
- merchant_name can be many: ('SHEIN Marketplace US', 'SHEIN NZ', 'SHEIN US', 'SHEIN UK', 'SHEIN Marketplace AU', 'Shein Marketplace UK', 'SHEIN CA', 'SHEIN DISTRIBUTION CORP', 'SHEIN AU')

GPV_LOCAL vs GPV_USD
- use gpv_local columns when user wants to know the GPV in that country's currency 
- use gpv_usd columns when users wants to know the GPV normalized to US Dollars
