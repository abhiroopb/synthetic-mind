Table Types:
- Analytics: These tables are curated for analysis and should be your primary uses. 
- Production: These tables are replicated data from your product's APIs and services. This data is not curated for analysis. Only use Production tables if there is no similar analytics table 
- Event: These tables track user initiated events such as clicks, taps, and views on your company's web and mobile software products. These tables contain a variety of events and require explicit filters on event descriptions and dates. 

Date Filters: 
- Always apply a date filter to a query. 
- Use the query_expert_search tool to find the most common date filter for that table 
- If the user does not request a date filter, suggest one
- Dimension tables are an exception. Often they do not have date fields to filter on.

Column Values: 
- Do not invent the values of a column. Always ensure accuracy based on query_expert_search tool or from the user input
- Use query_expert_search tool to find potential values for a particular column on a table by using the table_names parameter and the column name as the search text with other relevant keywords 
- If column values are unknown, execute a simple query selecting the distinct column_name with a short date period. Or an aggregate function
    - For example, "SELECT DISTINCT feature_key, category, sub_category FROM YOUR_SCHEMA.FEATURE_DIM" 
    - For example, "SELECT country_code, count(distinct customer_id) as customers from YOUR_SCHEMA.DIM_CUSTOMER group by 1 order by 2 desc Limit 100" 
- If a dimension table exists to discover column values, then use that one over a larger, transactional or aggregated table 
    - For example, to understand PRODUCT_NAME values, use YOUR_SCHEMA.DIM_PRODUCT instead of querying for PRODUCT_NAME within YOUR_SCHEMA.FACT_SUBSCRIPTION_STATES

Brand A Customers <> Brand B Customers 
- Cross-brand mapping tables exist to link customer accounts across different product lines 
- These tables live in a dedicated cross-mapping database/schema 

Active:
- Active customers generally mean processing payments in the last 91 days or windowed to 91 days of the analysis unless specified by the user
- Active for Products (Subscription products) and features often mean 30 days or windowed to a month unless specified by the user
