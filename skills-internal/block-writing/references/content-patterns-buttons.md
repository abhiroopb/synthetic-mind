# Content Patterns: Buttons

## Primary Action Buttons (CTA)
Template: `[Verb] + [Object/Context]` — ~20 chars — bottom of screen, end of flow

```
✅ "Complete order", "Send payment", "Add money", "Save changes"
❌ "Submit", "OK", "Done", "Finish", "Next" (without context)

Cash App: "Send payment", "Add money", "Withdraw funds", "Complete transfer"
Square: "Process payment", "Complete sale", "Finish setup", "Save settings"
Afterpay: "Split payment", "Complete purchase", "Confirm plan"
Tidal: "Start listening", "Create playlist", "Download album"
Bitkey: "Send bitcoin", "Secure wallet", "Complete setup"
```

## Secondary Action Buttons
Template: `[Alternative action]` or `[Dismissal]` — ~20 chars — paired with primary

```
✅ "Not now", "Cancel", "Save as draft", "Skip for now", "View details"
❌ "Maybe later", "No thanks", "Don't want"

Cash App: "Not now", "Use different card", "Skip verification"
Square: "Save as draft", "Try different method", "Contact support"
Afterpay: "Pay in full", "Choose different plan"
```

## Compact Buttons (Inside Components)
Template: `[Single verb]` or `[Verb + noun]` — ~15 chars — inside cards, lists, forms

```
✅ "Edit", "Save", "Delete", "Add card", "View more", "Try again"
❌ "Edit payment method" (too long), "Edit the card" (has article)

Cash App: "Edit", "Remove", "Add card", "View history"
Square: "Edit", "Delete", "Add item", "View report"
```
