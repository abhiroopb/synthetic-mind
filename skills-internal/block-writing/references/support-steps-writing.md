# Writing Steps for Support Content

## When to Use Steps (vs. Bullets)
Use **numbered steps** when ANY of these are true:
- Describing a workflow, troubleshooting guidance, or instructions
- More than 3 distinct actions are presented
- Information is sequential (changing order loses meaning)

Use **bullet points** when fewer than 3 steps, or when order doesn't matter.

Don't include assumed "foundational" steps (e.g., "turn on your phone", "open the app").

## Step Guidance

**Tone**: Technical, precise, instructional, and clear
**Language**: Plain, direct; no jargon; 6th–8th grade reading level; present tense
**Format**:
- One action per step — separate context from the action itself
- Bold UI elements (buttons, menu items, fields, system messages)
- Use exact names as they appear in the product (no paraphrasing)
- Start each step with a strong verb
- Sentence punctuation; no exclamation marks

✅ "Close" ❌ "Terminate" — use everyday verbs

**Decisions within steps**: If a decision is needed, first consider drafting separate step sets. If it must stay in one flow, nest the decision under the related action.

Example:
> Check for unauthorized transactions.
> - If found: help the customer open a dispute claim.
> - If not found: continue with the steps below.

## Hierarchy
- Aim for one level; two levels max (for multi-surface instructions)
- If hierarchy exceeds three levels → consider creating a new article

## Example: Well-formatted steps
> Create a new CF1 case
> 1. Open the Case tab in Sprinklr.
> 2. Click the Edit icon.
> 3. Navigate to SFDC and enter customer details:
>    - Customer C-Token
>    - Customer email address
> 4. Add case information in the Description field.
> 5. Click Save.
> 6. Click Send. (Clicking Send keeps the case in your queue.)
