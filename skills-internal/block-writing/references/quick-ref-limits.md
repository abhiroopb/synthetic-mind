# Component Character Limits

## Strict Limits (Must Not Exceed)

| Component | Character Limit | Context |
|-----------|----------------|---------|
| Button Compact | ~15 characters | Inside containers, cards, lists |
| Button Default | ~20 characters | Standalone buttons in screen content |
| Button CTA | ~20 characters | Bottom of screen, end of flow |
| Input Field Labels | ~25 characters | Form field labels (1-3 words) |
| Modal Titles | 25-40 characters | Modal header text (5-7 words) |

## Flexible Limits (Best Practice Guidelines)

| Component | Character Limit | Context |
|-----------|----------------|---------|
| Input Placeholder | ~25 characters | Format examples only |
| Input Helper Text | ~100 characters | Supporting info (2 lines max) |
| Modal Body | ~100 characters | Modal description (2-3 lines) |
| Input Card Content | ~130 characters | Total content per card |
| Toast Header | 75 characters | 50 characters if using body text |
| Toast Body | 75 characters | Additional context |

## No Fixed Limits (Keep Brief and Scannable)

| Component | Guideline |
|-----------|-----------|
| Cell Labels | Keep short for scanning |
| Checkbox Labels | Brief but descriptive |
| Toggle Labels | Short and clear |
| Radio Labels | Single line preferred |
| List Items | Scannable length |
